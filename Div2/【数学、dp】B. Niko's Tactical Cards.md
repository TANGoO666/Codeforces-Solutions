https://codeforces.com/contest/2173/problem/B

## 题目描述：

> Niko 正在玩一个游戏。他最初的分数是 $0$，分数用 $k$ 表示   
> 游戏有 $n$ 个回合, 在第 $i$ 个回合中，他可以选择一张 红牌 或 一张 蓝牌  
> 红牌上的整数为 $a_{i}$ ，蓝牌上的整数为 $b_{i}$  
> 如果他选 红牌 ，他的得分就会变成 $k - a_{i}$  
> 如果他选 蓝牌 ，他的得分就会变成 $b_{i} - k$  
> 之后，游戏进入下一轮，如果是第 $n$ 轮，则游戏结束  
> 求 Niko 在游戏结束时能获得的 **最大分数**  

## 核心思路：

此题不能单独维护一个最大值贪心求解

因为 蓝牌 是神牌，它可以实现分数逆转（如果 $k$ 是一个很小的负数，那么减一下直接起飞）

此题应维护一个 最大值 $mx$ 和 最小值 $mn$ , 利用 **动态规划** 求解

$mx$ 代表： 当前回合选牌前可能获得的 最大值

$mn$ 代表： 当前回合选牌前可能获得的 最小值

我们可以根据 红牌 与 蓝牌 的分数运算规则，进行数学推导

首先， 一定有：

$$
mn ≤ mx
$$

选择红牌（$k - a_{i}$）：

$$
mn - a_{i} ≤ mx - a_{i}
$$

这意味着，选择红牌， **最小值** 一定不可能会超过 **最大值** 

选择蓝牌（$b_{i} - k$）：

由于 $k$ 前面多了个负号，因此：

$$
-mn ≥ -mx
$$

$$
b_{i} - mn ≥ b_{i} - mx
$$

这意味着，选择蓝牌，**最小值** 有可能会超过 **最大值** 

也就是说，我们得到了两个关键结论：

- 选择红牌， **最小值** 一定不可能会超过 **最大值**  （最小值永远无法翻身）
- 选择蓝牌，**最小值** 有可能会超过 **最大值** （这就是为什么，此题不能单独维护一个最大值贪心求解）

有了这两个结论，这题就很好解了

因为在第一个回合选牌前，分数是 $0$ , 所以初始化 `mx = 0, mn = 0` 

然后动态维护 $mx$ 和 $mn$ 

**需要注意的是**： 每次动态维护的时候，额外声明 $mx$ 和 $mn$ 的拷贝变量，以防动态更新 $mx$ 或 $mn$ 的时候，新值覆盖了旧值导致出错

## AC Code：

```cpp
#include <bits/stdc++.h>
using ll = long long;
using namespace std;

int main() {
    ios::sync_with_stdio(0);
    cin.tie(0);
    cout.tie(0);
    int t; cin >> t;
    while(t--) {
        int n; cin >> n;
        vector<ll> a(n + 1), b(n + 1);
        for(int i = 1; i <= n; i++) cin >> a[i];
        for(int i = 1; i <= n; i++) cin >> b[i];
        ll mx = 0, mn = 0;
        for(int i = 1; i <= n; i++) {
            ll mx2 = mx, mn2 = mn;
            mx = max(mx2 - a[i], b[i] - mn2);
            mn = min(mn2 - a[i], b[i] - mx2);
        }
        cout << mx << "\n";
    }
}
```
